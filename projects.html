Projects

Personal Website

I began this project to keep a portfolio of my work and to improve my web development skills. Starting in August 2014, I discovered an excellent library, Twitter Bootstrap, which I've used to create my website from the ground up. The website uses HTML5, CSS3, and JavaScript. It also uses jQuery for some cool visual effects and it is completely responsive. Over the course of the project, I've put a good amount of thought into the design as well as maintaining the website's mobile friendliness and browser compatibility.

Briscola Statistics Tracker

This is a simple Android app that can elegantly record all the data pertaining to a single game of Briscola (a trick based card game similar to Euchre). It keeps track of who is playing, who won the bid, what card he or she called, and so on. The app also allows going back and changing information in an intuitive way. I started this app because of the folks over at Sony (SNEI) in Waterloo. They play this game everyday during lunch and I had a blast playing with them when I worked there. During this project, I learned about the Android activity life-cycle and how to create clean UI with XML.

Project Euler

When I'm bored and just want to do something that requires very little setup, I start working on a problem from Project Euler. Project Euler is a website with hundreds of mathematic and computer programming problems. I've done about 30 problems to date, and all of my solutions are in C++. My goal for every problem is to solve it with the most efficient algorithm, rather than the simplest. I find it much more rewarding to come up with an intelligent solution than to just brute force every problem. I also strive to make each of my solutions flexible in terms of an input variable. For example, instead of finding the 10,000th prime number, I would create a program to find the n-th prime number and use it to find the prime number I want. All in all, I've sharpened both my C++ skills and my algorithm skills through Project Euler.

Minesweeper

Back when I was in grade 12, I played a fair bit of Minesweeper on Windows. Displeased with the lack of certain features, I decided I would create my own Minesweeper app in C++. I started the project by coding just the game logic and having the game's interface be through the command line. As a result, I had a working product fairly quickly. Then I began adding features, like a GUI, settings, and high scores, which would eventually finish the project. I used a library called SFML to perform OS specific tasks like creating windows and drawing things. This means my app works in Windows, OS X, and Linux. I'm very happy with the finished product and I'm particularly fond of the statistics that it tracks.

Dominion

Dominion is a very fun deck-building card game. However, while playing the game, you end up doing a lot of shuffling. Thus, I decided to code the game in C++, and to make my life a little easier, the game runs within a console. This project was interesting from a design perspective because there is a vast amount of cards with a wide variety of behaviours. Properly designing the game engine would make the rest of the code much cleaner and simpler. I also wanted to take into account the possibility of implementing cards from the game's several expansions. In the end, I do believe the design I came up with was an effective one. This game also uses SFML so it is cross-platform. Another big thing about this project is that it completely supports multi-player and single-player through a client-server model. I like what I did with this project and I might end up revisiting it sometime later in the future.
